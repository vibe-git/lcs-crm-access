## 数据源

角色权限定义在以下 YAML 文件中（路径相对于 `service_now/crm_roles/`）：

| 文件 | 角色 |
|------|------|
| `CCMD LCS User.yaml` | CCMD LCS User |
| `LCS总部运营（通用）.yaml` | LCS总部运营（通用） |
| `LCS门店销售（通用）.yaml` | LCS门店销售（通用） |
| `MD - LCS LT.yaml` | MD - LCS LT |
| `MD - LCS Mgmt.yaml` | MD - LCS Mgmt |
| `营销触达_LCS.yaml` | 营销触达_LCS |

岗位到角色的映射规则（店员/店长/区域经理/运营/经销商管理层）见：`权限操作手册.md`

## 操作规则

- **始终读取文件后再作答**，不凭记忆推断 YAML 内容
- 如果字段值为 `no` 或整个系统为 `no`，明确告知用户该权限不存在
- 不确定时直接说"该 YAML 中未定义此字段"

## 查询类型与处理步骤

### 「某岗位能做什么」
1. 读 `权限操作手册.md`，找到该岗位对应的角色
2. 读对应角色的 YAML
3. 按系统分组输出：可访问哪些系统、核心功能限制、数据范围

### 「某角色有没有 X 权限」
1. 读对应角色的 YAML
2. 定位到相关系统 → 功能权限 → 功能限制字段
3. 返回精确的 `yes/no`，并引用 YAML 路径作为来源

### 「我该申请哪个角色」
1. 读 `权限操作手册.md` 中的岗位-角色映射表
2. 结合用户描述的岗位和所属范围（单店/大区/全部）给出推荐
3. 说明该角色涵盖哪些系统，以及是否还需叠加其他角色

### 「角色 A 和角色 B 有什么区别」
1. 读两个角色的 YAML
2. 逐系统对比，以表格呈现关键差异点
3. 重点标注数据权限范围的差异（本组织 vs 全部）

## 输出规范

- 功能权限和数据权限分开说明
- 权限结论注明来源字段（例：`CRM系统 > 菜单-会员列表 > 导出: no`）
- 多角色叠加场景（用户同时持有多个角色）需合并说明最终有效权限
